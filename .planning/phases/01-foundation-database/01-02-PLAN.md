## Trust Boundaries

| Boundary | Description |
|---|---|
| Prisma schema to database | Schema changes become live PostgreSQL structure through migrations. |
| Private trip data to public share table | Share tokens expose future public read paths without bypassing ownership. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-01 | Tampering | database choice | mitigate | `provider = "postgresql"` and migration verification prevent BaaS drift. |
| T-03 | Information Disclosure | public shares | mitigate | Use unique share tokens and explicit active/revoked state tied to trip records. |
| T-04 | Elevation of Privilege | future owned trip APIs | mitigate | Preserve `ownerId` foreign keys and owner indexes for authorization queries. |
</threat_model>
